# Visual Direction v1

## Purpose

Defines the visual direction for FaeBook so implementation stays cohesive.

## Locked Decisions

- the app stays in dark mode
- the app should feel polished and less like a dev build
- the overall UI should feel fae-themed
- the investigation board should feel more like a murder-board surface than the rest of the app
- the fairy logo and FaeBook branding stay part of the product identity

## Overall Tone

FaeBook should feel:

- dark
- elegant
- magical
- intentional
- slightly mysterious
- clean enough for repeated play use

It should avoid:

- generic dashboard blandness
- dev-tool visual clutter
- harsh unfinished spacing
- bright utilitarian accents that break the setting tone

## App-Wide Palette Direction

Suggested palette family:

- moonlit lavender
- dusky plum
- soft rose
- muted teal
- moss-green accents
- warm parchment and cork accents where thematic
- berry-red for destructive actions

## Core UI Surfaces

### App shell

- dark background
- soft panel layering
- clear visual hierarchy
- polished drawer and utility row

### Drawer

- restrained, elegant styling
- active route state clearly visible
- bottom utility row feels anchored and intentional

### Dashboard

- cleaner and more polished than the board
- modular card system
- readable spacing
- strong dark-mode contrast

### NPC Directory

- portrait cards feel clean and premium
- metadata chips feel consistent
- search and filter controls feel integrated

## Investigation Board Direction

This page should feel like a real physical evidence board.

### Keywords

- corkboard
- pinned photos
- sticky notes
- layered paper
- hand-built theory space
- tactile mystery surface

### Visual ideas

- sticky notes with selectable colors
- pinned NPC photo cards
- subtle paper shadows
- warm textured board background
- evidence-style lines between nodes
- bottom status text that feels like part of the board frame

### Board controls

- floating controls inside the board
- compact and purposeful
- do not overwhelm the board surface

## Portrait Treatment

- transparent-background portraits preferred
- no aggressive white background blocks
- card framing should feel soft and intentional
- portraits should sit naturally on dark surfaces

## Maps Direction

- cleaner than the board
- legible and functional
- map controls should feel polished and simple
- pin UI should stay readable without clutter

## Typography and Spacing

- keep headings clear and elegant
- use the FaeBook brand mark consistently
- avoid cramped spacing
- preserve readability first

## Motion and Interaction Feel

- subtle transitions
- no heavy flashy animation required
- interactions should feel smooth and deliberate
- destructive actions should feel clearly separated

## Release Goal

The finished UI should feel like a real campaign companion product with a strong identity:

- dark fae shell
- tactile murder-board center
- polished navigation
- clean dashboard
- readable maps
- consistent role-aware admin surfaces
